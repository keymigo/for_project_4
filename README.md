# Тестирование и оптимизация гипотез для улучшения бизнес-процессов

## Цель проекта
Выбрать наиболее перспективную гипотезу для улучшения бизнес-процессов на основе количественной оценки потенциального эффекта и затрат, используя фреймворки ICE и RICE.

## Задачи проекта
- Проанализировать набор гипотез с заданными параметрами.
- Рассчитать приоритет гипотез с использованием метрик ICE и RICE.
- Сравнить результаты приоритизации разными подходами.
- Выбрать гипотезу с наибольшим потенциалом для дальнейшего тестирования.

## Методология и инструменты
Проект выполнен на языке **Python** с использованием следующих библиотек:
- **pandas**, **numpy** — обработка и анализ данных;
- **matplotlib** — визуализация результатов.

Формат проекта — **Jupyter Notebook (`.ipynb`)**.

## Источники данных
Входные данные представлены в виде файла с набором гипотез, для каждой из которых заданы следующие параметры:
- **Reach** — охват пользователей;
- **Impact** — ожидаемое влияние на ключевые метрики;
- **Confidence** — уверенность в оценке гипотезы;
- **Effort** — трудозатраты на реализацию.

## Приоритизация гипотез
Для выбора гипотезы были использованы два фреймворка:

- **ICE** — учитывает Impact, Confidence и Effort;
- **RICE** — дополнительно учитывает Reach, позволяя точнее оценить масштаб потенциального эффекта.

Для каждой гипотезы были рассчитаны соответствующие значения ICE и RICE, после чего выполнено ранжирование.

## Результаты и выводы
- Приоритизация с помощью ICE и RICE может приводить к разным результатам.
- Использование RICE позволяет учитывать не только силу эффекта, но и масштаб аудитории.
- Выбранная гипотеза обладает наилучшим соотношением потенциального влияния и затрат и рекомендуется для дальнейшего тестирования.

## Ограничения
- Оценки параметров гипотез носят экспертный характер.
- Анализ не учитывает возможные внешние факторы и риски реализации.
- Результаты требуют последующей проверки через эксперимент или A/B-тестирование.
